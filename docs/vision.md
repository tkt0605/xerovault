# Xero Vault — ビジョン

> 本ドキュメントはコード・既存機能から逆算した現状のドラフトです。プロダクトの意図と食い違う点があれば書き換えてください。

## コンセプト

個人の目標管理アプリの多くは「立てて終わり」になりがちで、達成の可否を自己申告に委ねるため曖昧な目標(「頑張る」「継続する」)が量産される。Xero Vaultは、目標をグループ内の**相互投票による合意形成**で達成扱いにすることで、次の2つを強制する。

1. **曖昧な目標を許さない** — 期限・担当者を明示した「具体的な目標」は、そうでない目標より高く評価される(`docs/score-design.md`参照)。
2. **自己申告を許さない** — 達成の可否はグループメンバーの投票(YES/NO)によって決まり、本人だけでは達成を宣言できない。

## 誰のためのアプリか

少人数(サークル・プロジェクトチーム・自習グループなど)で、互いの進捗に責任を持たせ合いたいグループ。

## コア体験

1. グループ(Group)を作り、メンバーを招待する
2. グループ内でゴール(Goal)を立てる。期限と担当者を設定するほど「具体的な目標」として扱われる
3. メンバーがゴールの達成についてYES/NO投票する。投票率が閾値(90%)を超えると達成扱いになる
4. 達成したゴールはグループのスコアに加算され、連続達成でストリークボーナスが付く
5. 公開グループはランキングで他グループと比較できる

## 現状のスコープ外(将来検討)

- 期限切れ・未達成に対するペナルティ(`docs/score-design.md`で設計中)
- リアルタイム通知(投票・チャットの即時反映)
- 本番デプロイパイプライン
