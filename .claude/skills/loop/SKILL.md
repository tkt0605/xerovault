# ブレインストーミング・ループ

対象テーマ: $ARGUMENTS

このコマンドは「発散→収束→次の一手」を1ラウンドとして、複数ラウンド繰り返す。同じ発想法を繰り返さず、ラウンドごとに視点を変えてアイデアが尽きないようにする。

## 実行ルール

1. **ラウンド数の決定**: 引数にラウンド数があればそれを使う。なければ3ラウンドを既定値とする。
2. **前提の確認**: 会話内にすでに出ているアイデアがあれば重複させず、それらを土台にする。テーマが曖昧な場合は想定を1行で明記してから進める(過度な質問はしない)。
3. **ラウンドごとに手法をローテーション**:
   - ラウンド1: 自由発散(判断保留、量を出す)
   - ラウンド2: SCAMPER
   - ラウンド3: アナロジー思考(異業種・自然界からの借用)
   - ラウンド4: 逆転の発想(最悪のシナリオから反転)
   - ラウンド5: 制約注入(予算ゼロ/10倍規模/禁止ワードなど)
   - 6ラウンド目以降はこのローテーションを繰り返す
4. **各ラウンドの出力形式**:
   - 見出しに「ラウンドN: [使用した手法]」と明記
   - アイデアを8〜12個列挙(短く、判断保留)
   - その後、それまでの全ラウンドのアイデアを合わせて簡潔にクラスタリングし、現時点の有望株を3つほど太字でハイライト
   - ラウンドの最後に、次ラウンドで何を掘り下げるか一言添える
5. **停止条件**: 指定ラウンド数を終えたら自動的に停止し、最終まとめに移る。ユーザーが「続けて」「もう1周」と言えばローテーションの続きから次のラウンドを行う。ユーザーが「ここで終わり」と言えばすぐ最終まとめに移る。
6. **最終まとめ**(最後のラウンドの後に必ず実施):
   - 全ラウンドを通じての上位5個のアイデア
   - それぞれの一言サマリーと、新規性/実現しやすさ/インパクトの目安
   - 次の一歩(検証方法、プロトタイプ、必要なリソースなど)の提案

## トーン

- 各ラウンドの発散パートでは絶対に否定・評価しない。
- 収束パートでは率直に「弱い」「重複している」も指摘してよい。
- ユーザーへの確認は最小限にし、テンポよくラウンドを進める。
