# 現在の状況確認

現在のリポジトリの変更内容を表示し、各変更にそってリスト化します。

- ステージング済みの差分: !`git diff --cached`


# 指示

上記の変更内容を確認した上で、ユーザーのリポジトリに対して以下のステップに則って実行してください。

### 1. 変更内容の確認

- 変更内容を確認し、ワーキングツリーのステータスを確認。その中に、`.env`などの環境変数ファイルやAPIキー、その他の秘密情報が漏洩されていないかを確認してください。
- その確認が完了いたら、`git status`を実行して、ステージングされているファイルとされていないファイルの一覧を表示してください。

### 2. ファイルのグループ化と変更のステージング(git add)

- 変更内容（修正・変更・削除）を論理的なグループ（コンポーネント、機能等）に分類してください。
- 分類されたグループごとに、`git add <ファイル名>`を実行して、ステージングエリアに追加してください。

### 3. コマンド引数の確認とコミット（git commit）

- ユーザーがこのコマンドの引数（`$ARGUMENTS`）としてコミットメッセージを指定している場合は、その文字列を最優先でコミットメッセージとして使用してください。
- 引数がからにお状態の場合は、ステージングされた変更内容を確認し、変更内容に沿った簡潔でわかりやすいConventional Commits（例: `feat:`, `fix:`）に順守したメッセージを自動生成してください。
- `git commit -m "<コミットメッセージ>"` を実行して、ステージングされた変更をコミット。

### 4. リモートへの反映(git push)

- コミットが完了したら、現在のブランチを確認し`git push`を実行して、リモートリポジトリに反映してください。
- 全ての処理が完了したら、プッシュした内容の要約を表示してください。
