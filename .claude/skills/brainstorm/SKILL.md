# ブレインストーミング

以下のテーマについてブレインストーミングを行う: $ARGUMENTS

## 進め方

1. テーマ・目的・制約(予算、期限、対象者など)が曖昧でも、いったん自分なりの想定を1行で明記してから着手する。本質的に見当違いになりそうな場合だけ、質問を1つだけする。
2. **量を優先するフェーズ**: 最初は評価・却下をせず、思いつく限り多くのアイデアを出す(目安15〜20個)。突飛なもの、笑えるもの、非常識なものも歓迎する。
3. 複数の発想法を意識的に使い分ける:
   - SCAMPER(代用・結合・応用・修正・転用・削減・逆転)
   - アナロジー思考(全く関係ない業界・自然界からヒントを借りる)
   - 逆転の発想("どうすれば最悪の結果になるか"から反転させる)
   - 制約注入("予算ゼロなら" "10倍の規模なら" など)
   - "Yes, and"で他のアイデアに乗っかって発展させる
4. 出したアイデアをテーマ別にグルーピングする。
5. それぞれに簡潔に付す: 新規性・実現しやすさ・想定インパクト(高/中/低)。
6. 最後に上位3〜5個を選び、有望な理由と次の一歩(試作、検証方法など)を提案する。

## トーン

- 早期に否定しない。「でも」より「そして」でつなぐ。
- 質より量を先に。収束は後回し。
- 実現可能性を気にしすぎない(それは後工程で評価する)。
